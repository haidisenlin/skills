# 季度持续迭代更新（Step 4）

---

**运行原则：** PDCA飞轮 × 文档飞轮加速机制。每季度补充新数据和新发现，更新飞书云文档，然后用本技能触发AI重新评估。随着文档积累越来越丰富，AI给出的建议会越来越精准——这是文档飞轮的持续加速机制。

**参考来源：** PDCA循环（Deming）· 敏捷回顾（Retrospective）· PSI漂移监控 · OKR季度复盘（John Doerr）

**建议频率：** 每季度执行一次；若发生以下情况可提前触发：
- 模型精度下降超过阈值（PSI>0.25）
- 发生重大质量事故
- 关键角色变动（PE/AIE离职）
- 管理层要求重新评估项目价值

---

## 你的任务：收集季度进展，重新评估并更新计划

### 第一步：收集本季度信息

向用户说明迭代更新的价值，并引导用户在飞书新建文档来提供更新信息：

**说明文字：**

```
欢迎进行本季度AI项目迭代更新！

距离上次制定计划已经过去了一段时间。本次更新将帮助你：
✅ 重新评定六维准备度（看哪些维度提升了）
✅ 调整剩余行动计划优先级
✅ 识别新解锁的AI方法
✅ 确定下季度最重要的补充工作

请在飞书新建文档来提供更新信息。
```

⚡ **[必须调用 research-template]** 使用 read_file 读取 research-template 技能的 SKILL.md（路径见系统提示插件 Skill 列表），然后按以下参数执行：

```
topic: "工厂AI项目季度PDCA迭代回顾 - 行动进展与准备度更新"
scenario: [用户当前的项目进展、已完成行动、KPI变化]
industry: [用户行业]
doc_name: "[厂区代码]-[项目简称]-季度迭代输入-[今日日期]"
purpose: "收集本季度行动完成情况、维度升级数据和KPI进展，为迭代评估提供输入"
```

⚠️ 此步骤必须通过 read_file 读取 research-template/SKILL.md 执行，不能用飞书MCP工具或直接输出模板替代。

收到子技能返回的模板后，发给用户并说明：

```
这是季度迭代更新的信息收集步骤。

请在飞书新建文档，按返回的模板填写本季度的：
- 行动完成情况（哪些完成了、哪些遇到阻碍）
- 六维维度的更新（评级有无上升）
- KPI进展（当前值 vs 基线 vs 目标）
- ROI和风险变化

完成后粘贴链接，我将重新评估六维准备度、识别新解锁的AI方法、更新行动优先级。
```

**等待用户提交飞书链接或内容后，进行迭代分析。**

## 低置信度处理原则

用户提供信息不完整时：

- **缺少上次评级：** 说明"由于无法获得上次评级数据，本次以当前信息重新建立基线"，然后按照 `/sunny-ai-project:readiness-diagnosis` 的标准输出全新评级
- **KPI数据不全：** 将对应指标标注为"[数据待补充]"，不影响其他部分输出
- **某维度无新进展：** 维持上次评级，在报告中注明"本季度无实质变化，评级保持[X]级"
- **行动完成情况不清晰：** 只分析用户明确提供的部分，其余标注"[请在下次迭代时补充]"

>
> 接下来可以：
> 1. 根据更新后的行动计划，向Sponsor汇报（参考月报模板：`/sunny-ai-project:execution`）
> 2. 如果六维评级有显著提升，可重新启动 `/sunny-ai-project:methodology`，获取更新后的AI方法推荐
> 3. 若准备度综合评级已达4级，可启动 `/sunny-ai-project:action-plan` 更新完整四阶段计划
