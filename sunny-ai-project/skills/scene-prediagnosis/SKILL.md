# 场景预诊断（Step 0-A）

---

## 第一轮：了解场景基本情况

如果用户来自场景发现（已有场景确认描述），直接跳到第二轮使用该描述。

否则，**一次只问一个问题**，按以下顺序：

1. 先问：公司/行业类型，以及改善领域属于哪类（A-J或K）
   ```
   A-生产效率 / B-质量管理 / C-设备可靠性 / D-精益流程
   E-供应链 / F-成本管理 / G-能源 / H-安全 / I-人才 / J-数字化 / K-自定义
   ```
2. 收到回复后，再问：具体是哪个工序？打算用什么AI来改善？
3. 收到回复后，再问：这个工序现在最大的痛点是什么？最好有具体数字

收到三个问题的回答后，进入第二轮。

**K路径（自定义场景）**：如果用户选择K，在继续之前先：
- 判断最接近哪类管理改善领域，给出业界最权威的目标定义方法论（TQM/TPM/Six Sigma等）
- 给出该场景的标准KPI（含公式和计量单位）、基线数据要求、世界级基准值
- 请用户确认方向，再进入第二轮

---

## 第二轮：输出场景识别 + 启动飞书准备

基于第一轮信息，输出以下内容，然后**停下来等用户确认**：

```
【场景识别结果】

AI调优类型：[视觉AI / 工艺参数优化 / 预测性维护 / 流程分析 / 混合型]

核心技术挑战：[2-3句话，说明这个场景的主要难点]

最关键的准备度维度（按重要性排序）：
  第1位：[维度名] — [为什么对这个场景最关键]
  第2位：[维度名] — [原因]
  第3位：[维度名] — [原因]

---

📋 接下来我们将逐步填写六维飞书文档。

⚠️ 所有信息请通过飞书云文档录入，不要只在对话框里回复。
   原因：飞书文档可以多人协作、长期积累、AI重复引用——这是"文档飞轮"的基础。

飞书文档统一命名规则：
  [厂区代码]-[项目简称]-[维度名称]-[日期]
  例：YC-AOI视检-数据准备度-20260412

我们从【最关键维度】开始。准备好了吗？回复"开始"继续。
```

---

## 第三轮起：逐维度引导（每次一个维度）

每个维度的完整流程：

**步骤1** — 告知当前进度，然后使用 Agent 工具调用 `sunny-ai-project:research-template` 子技能，生成该维度的飞书文档模板，并说明为什么要收集这些内容

**步骤2** — 等用户把飞书文档链接或填写内容粘贴回来

**步骤3** — 简单确认收到，告知下一个维度是什么，问是否继续

按第二轮识别的重要性顺序，逐一引导六个维度。

## 中途信息不足的处理

用户填写某个维度时，如果信息明显缺失：

- **不要要求用户补充完整再继续** — 先记录已有信息，标注「低置信度，建议后续补充」
- 告知用户缺失信息对诊断的影响程度（高/中/低影响）
- 继续引导下一个维度，不阻塞流程

例如：
「②数据准备度已记录，目前数据量信息缺失，会影响数据准备度评级的精确度（中等影响）。这部分可以在准备度诊断之前补充。我们继续看③技术互联度，已完成 2/6。」
