# 飞书云文档 · 文档飞轮规范

## 核心理念

文档越丰富 ──→ AI 理解越深入 ──→ 建议越精准可落地 ──→ 改善效果越好
     ↑                                                          │
     └────────── 团队更有动力持续补充和更新文档 ←──────────────┘

每份写入飞书的说明、数字、截图，都在帮 AI 更准确地理解工厂。

## 文档命名规范

```
[厂区/事业部代码]-[改善项目简称]-[内容模块]-[YYYYMMDD]
```

示例：`SUNNY光电-车载模组AI改善-数据准备度评估-20260412`

## 每份飞书文档必要结构

- **文档头**：填写人 | 部门 | 创建日期 | 最后更新 | 版本号
- **H1 大标题**：模块名称
- **H2 子标题**：每个问题组一个 H2
- **正文原则**：
  - 每个要点至少 2-3 句完整句子
  - 有数字必写数字；"大概""可能"后必须补依据
  - "我们已经做了"后面要跟"结果是___，时间是___"
- **图片与附件**：GR&R 报告、SPC 图、系统架构图直接拖入飞书

## 好文档 vs 差文档

| 差（AI 无法利用）| 好（AI 能深度分析）|
|---|---|
| "GR&R 做了，结果还行" | "GR&R 结果 22%，按 AIAG MSA 标准属边界可用，主要误差来自操作员间差异" |
| "良率不太好" | "2025Q4 车载模组 3# 产线 FPY 平均 89.3%，主缺陷：镜片脏污 47% + 组装偏移 31%" |
| "换型比较慢" | "换型从末件到首件合格平均 47 分钟，夹具对准 25 分钟（53%），SMED 目标 < 10 分钟" |
| "管理层支持" | "Owner 为制造部总监王 X，绩效考核含 FPY 提升目标，已批 ¥50 万专项预算（附审批截图）" |

## 持续更新机制

- 每次有新数据或结论，文档末尾追加"📅 [YYYYMMDD] 更新：[变化]"
- **不要删改历史内容**，保留演进轨迹
- 每季度至少检查一次，确认内容反映当前实际状态

## AI 自动生成的飞书文档规范（各 skill 内联创建时必遵）

各 skill 的 SKILL.md 已定义「内联问卷生成协议」，由 skill 直接调用 `deep_research`（深度研究）+ 飞书 MCP `sunny_ai__feishu_create_doc`（创建飞书云文档）创建飞书问卷文档。生成文档时必须满足：

| 规范项 | 要求 |
|---|---|
| 字段标注 | **只用**🟡 推荐填 / ⚪ 选填两档；**禁用**"必填/必选/强制" |
| 比例 | 推荐填 : 选填 ≈ 6:4 |
| 量化占比 | 数字 / 比例 / 计数字段 ≥ 50% |
| 章节长度 | 每章节 ≤ 8 行主要项目 |
| 字段说明 | 每字段括号内给格式 / 范围 / 单位 / 示例 |
| 示例行 | 每章节 1-2 行 *斜体示例*，标注"（示例，请替换）" |
| 末尾章节 | 必含「📌 填写说明」解释 🟡 / ⚪ 含义 |
| 命名 | `[厂区]-[项目简称]-[内容]-[YYYYMMDD]` |
| 方法论注明 | 文档头部写"方法论来源：[2-3 个真实框架]"，不虚构引用 |

**设计理由**：制造现场 workshop 有时间限制，所有字段应允许留空。🟡 缺了 AI 会在最终产物标"[低置信度：X 未填]"但不阻塞流程。

## 当用户询问如何写飞书文档时

告知用户：
1. 命名：`[厂区]-[项目简称]-[模块]-[YYYYMMDD]`
2. 文档头必填：填写人 / 部门 / 日期 / 版本
3. 按 H2 结构分章节，每章节至少 3 句叙述
4. 有数字必写数字，"大约""可能"后补来源
5. GR&R 报告、SPC 图等**直接拖入**
6. 写完把链接粘贴回对话

> 💡 **多数情况下无需用户手动新建文档**——各 skill 内联协议会自动调用飞书 MCP `sunny_ai__feishu_create_doc`（创建飞书云文档）生成带模板的问卷飞书文档给用户填。
