# 飞书云文档 · 文档飞轮规范

## 核心理念

文档越丰富 ──→ AI 理解越深入 ──→ 建议越精准可落地 ──→ 改善效果越好
     ↑                                                          │
     └────────── 团队更有动力持续补充和更新文档 ←──────────────┘

每份写入飞书的说明、数字、截图，都在帮 AI 更准确地理解工厂。

## 文档命名规范

```
[厂区/事业部代码]-[改善项目简称]-[内容模块]-[日期]
```

示例：`SUNNY光电-车载模组AI改善-数据准备度评估-20260412`

## 每份飞书文档必要结构

- **文档头（顶部）：** 填写人姓名 | 所属部门 | 创建日期 | 最后更新日期 | 版本号
- **H1 大标题：** 模块名称
- **H2 子标题：** 每个问题组一个 H2，不要把所有内容挤成一段
- **正文原则：**
  - 每个要点至少 2-3 句完整句子，不写孤立词语
  - 有数字必须写数字；"大概""可能"后面必须补充依据
  - "我们已经做了"后面要跟"结果是___，时间是___"
- **图片与附件：** GR&R报告、SPC图、系统架构图直接拖入飞书，比复制文字更有价值

## 好文档 vs 差文档

| 差文档（AI无法有效利用）| 好文档（AI能深度分析）|
|---|---|
| "GR&R做了，结果还行" | "GR&R结果22%，按AIAG MSA标准属于边界可用，主要误差来自操作员间差异" |
| "良率不太好" | "2025Q4车载模组3#产线FPY平均89.3%，主缺陷为镜片脏污(47%)+组装偏移(31%)" |
| "换型比较慢" | "换型从末件到首件合格平均47分钟，夹具对准耗时约25分钟(53%)，丰田SMED目标<10分钟" |
| "管理层支持" | "项目Owner为制造部总监王X，绩效考核含FPY提升目标，已批复¥50万专项预算（附审批截图）" |

## 持续更新机制

- 每次有新数据或新结论时，在文档末尾追加"📅 [日期] 更新：[简述变化内容]"
- **不要删改历史内容**，保留演进轨迹——AI看到变化过程本身就是有价值的信号
- 每季度至少检查一次，确认内容仍反映当前实际状态

## 当用户询问如何写飞书文档时

告知用户：
1. 命名格式：`[厂区]-[项目简称]-[模块名]-[日期]`
2. 文档头必须有：填写人/部门/日期/版本
3. 按H2结构分章节，每章节至少3句完整叙述
4. 有数字的地方必须写数字，"大约""可能"后面补来源
5. GR&R报告、SPC图等截图直接拖入文档
6. 写完后把链接粘贴回对话框
