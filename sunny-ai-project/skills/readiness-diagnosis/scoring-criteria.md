# 六维准备度评分标准（1-5级）

## ① 战略价值度评级标准

| 级别 | 判定信号 |
|---|---|
| **1级** | 问题描述模糊（"良率不好"）；无量化基线；无明确Owner；无预算；没想过ROI |
| **2级** | 有优化方向但无具体数字；Owner是挂名的；预算"正在申请中"；ROI靠感觉 |
| **3级** | 问题量化清楚（有基线+目标+时间节点）；Owner有实权；有预算；ROI初步估算合理 |
| **4级** | 连接到公司战略优先项；Owner绩效与项目挂钩；有止损机制；ROI在保守假设下仍为正 |
| **5级** | 项目纳入公司OKR；成本与收益有系统追踪机制；有不行动代价的量化分析；成果可对外发布 |

## ② 数据准备度评级标准

### D1 数据质量与完整性
| 级别 | 判定标准 |
|---|---|
| **1级** | 未做MSA；缺失率>20%；无数据质量管理 |
| **2级** | GR&R>30%（不可接受）；缺失率10-20%；仅做过简单清洗 |
| **3级** | GR&R 10-30%（可接受）；缺失率<10%；有基本清洗流程 |
| **4级** | GR&R<10%（优秀）；缺失率<5%；有自动化数据质量监控 |
| **5级** | GR&R<5%；缺失率<1%；实时质量告警，异常自动拦截 |

### D2 X-Y可追溯性
| 级别 | 判定标准 |
|---|---|
| **1级** | X与Y存储在完全独立系统，无关联字段 |
| **2级** | 有时间段或班次对齐，关联率<60%；需大量人工处理 |
| **3级** | 关联率60-80%；有共同ID（SN/批次）；时差可控 |
| **4级** | 关联率≥90%；自动关联，时差<±5分钟；有验证机制 |
| **5级** | 关联率≥98%；全程自动化；实时数字孪生追溯 |

### D3 数据量与历史深度
| 级别 | 判定标准 |
|---|---|
| **1级** | 数据<3个月，样本量<1,000，缺陷样本<10例 |
| **2级** | 3-6个月，1,000-5,000样本；缺陷样本10-25例；工况单一 |
| **3级** | 6-12个月，5,000-50,000样本；缺陷样本25-50例；覆盖主要工况 |
| **4级** | 12-24个月，50,000+样本；缺陷样本50-100例；工况全面 |
| **5级** | >24个月，100,000+样本；持续积累机制；工况变化全记录 |

场景最低要求参考：
- 视觉检测AI：每类缺陷1,000-5,000张标注图像；标注错误率<1%
- 工艺参数优化：100+种参数组合记录；历史数据≥6-12个月
- 预测性维护：2-5年历史传感器数据；30-50+次真实失效事件；亚小时级采集频率
- 良率根因分析：50-100+生产批次；≥80%的缺陷批次有根因分类

### D4 标注质量与监督信号
| 级别 | 判定标准 |
|---|---|
| **1级** | 无结构化标注；Y值靠人工估算；无质量检查 |
| **2级** | 有标注但无标准；标注人员冲突率>20%；无复核 |
| **3级** | 有基本标注规范；冲突率<20%；有抽检机制（≥10%复核）|
| **4级** | 详细标注文档；Cohen's Kappa>0.8；系统化质量审计 |
| **5级** | 主动学习框架；边缘案例持续补充；标注质量持续追踪 |

### D5 稳定性与漂移感知
| 级别 | 判定标准 |
|---|---|
| **1级** | 工艺变更无记录；不知道数据是否过时 |
| **2级** | 有变更记录但未在数据中标记；数据混用新旧工艺 |
| **3级** | 主要工艺变更已在数据中标记；可分段训练；PSI已知 |
| **4级** | PSI<0.1；有漂移监控机制；模型性能自动追踪 |
| **5级** | 自动漂移检测+触发重训练；特征分布实时监控 |

*PSI参考值：<0.1=稳定，0.1-0.25=需监控，>0.25=显著漂移*

### D6 数据治理与可及性
| 级别 | 判定标准 |
|---|---|
| **1级** | 无数据字典；命名混乱；AI无法自主访问数据 |
| **2级** | 有部分字段说明（<50%覆盖）；命名不统一；访问需人工申请 |
| **3级** | >70%字段有说明；基本命名规范；AI有只读访问权限 |
| **4级** | >90%元数据覆盖；有数据血缘文档；API自动化访问 |
| **5级** | 完整数据目录；自动化血缘追踪；数据产品化；实时供给AI |

### ② 综合评级
| 综合级别 | 研判标准 | 推荐行动 |
|---|---|---|
| **1级** | D1-D6多数为1-2级 | 先建数据基础设施，12-18个月后重评 |
| **2级** | D1/D2/D3有2级但关键维度不足 | 制定数据补建计划，优先修复最低分维度 |
| **3级** | D1-D6平均≥3级，无单项<2级 | **最低启动线**：可以开始小范围PoC |
| **4级** | D1-D6多数≥4级 | 具备工业级AI部署条件 |
| **5级** | D1-D6全部≥4级，D3/D6达5级 | 数字孪生+自适应AI基础完备 |

## ③ 技术互联度评级标准

| 级别 | 判定标准 |
|---|---|
| **1级** | 设备无数字接口；数据只能人工读取抄写；系统完全孤岛 |
| **2级** | 有部分数字接口但无标准协议；数据导出需手动操作；延迟>1天 |
| **3级** | 主要设备有OPC-UA/MQTT等标准接口；MES/SCADA有数据；延迟<1小时 |
| **4级** | 设备数据实时流入；系统间API集成；IT/OT网络有受控互联；延迟<5分钟 |
| **5级** | 全程数字主线；实时数据湖；边缘计算；AI系统零延迟访问 |

## ④ 工艺可控度评级标准

| 级别 | 判定标准 |
|---|---|
| **1级** | 关键参数未识别；无GR&R；参数靠经验设定；无法实验修改 |
| **2级** | 有关键参数列表但未验证；GR&R>30%；参数修改需跨部门审批且周期>1周 |
| **3级** | 关键参数识别清楚；GR&R 10-30%（可接受）；有工程师权限的参数调整窗口 |
| **4级** | GR&R<10%；有Cpk监控；X→Y物理机理明确；参数调整有快速审批通道 |
| **5级** | 实时SPC管控；DoE验证的参数范围；AI系统具备闭环调参权限 |

## ⑤ 组织能力度评级标准

| 级别 | 判定标准 |
|---|---|
| **1级** | 无AI/数据能力；工艺知识纯靠口耳相传；IT支持极弱；各部门各自为政 |
| **2级** | 有IT支持但不懂ML；工艺文档残缺；过去有跨职能项目但困难重重 |
| **3级** | 有AI/数据工程师（或可外包）；关键工艺知识有文档；IT能配合搭建数据管道 |
| **4级** | 有T型人才（懂工艺+AI）；工艺知识系统化；跨职能协作顺畅 |
| **5级** | 内部AI卓越中心（CoE）；工艺+AI+IT完全融合；知识可跨工序复制 |

## ⑥ 管理变革度评级标准

| 级别 | 判定标准 |
|---|---|
| **1级** | 操作员/班长明确抵制；变更审批繁琐（>1个月）；AI被视为威胁 |
| **2级** | 观望态度为主；变更审批1-4周；高管支持但执行层阻力大 |
| **3级** | 关键班长/组长支持；变更审批<1周；有过成功推行改善的历史 |
| **4级** | 操作员主动参与设计；变更快速通道；AI辅助决策已有试验先例 |
| **5级** | AI推荐作为标准操作程序；自我改善的精益文化；知识分享激励机制 |
