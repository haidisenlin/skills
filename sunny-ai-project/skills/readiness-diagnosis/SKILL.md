# 准备度诊断（诊断交互 Step 1 + Step 1.5）

---

**方法论来源：** McKinsey Rewired AI准备度评估 · BCG AI at Scale启动门槛判定 · INCIT AIMRI成熟度分级 · WEF Lighthouse关键瓶颈识别（5M1E根因分解）

**前提：** 用户已完成阶段1（目标定义）和阶段2（六维资料收集）。

---

## 你的任务：基于用户提供的信息，输出完整诊断报告

### 第一步：信息收集确认

在正式输出诊断报告前，要求用户在飞书新建一份"诊断输入清单"文档：

⚡ **[必须调用 research-template]** 使用 read_file 读取 research-template 技能的 SKILL.md（路径见系统提示插件 Skill 列表），然后按以下参数执行：

```
topic: "工厂AI项目六维准备度诊断输入清单"
scenario: [用户已完成的目标定义 + 六维资料收集阶段]
industry: [用户行业]
doc_name: "[厂区代码]-[项目简称]-准备度诊断输入-[今日日期]"
purpose: "整合目标定义、六维资料链接和基线数据，为AI诊断提供完整上下文"
```

⚠️ 此步骤必须通过 read_file 读取 research-template/SKILL.md 执行，不能用飞书MCP工具或直接输出模板替代。

收到子技能返回的模板后，发给用户并说明：

```
这是诊断前的信息汇总步骤。

请在飞书新建文档，把以下内容整理到文档中：
- 阶段1的KPI矩阵和目标定义摘要
- 六维准备度文档的链接（已经在场景预诊断中填写过了）
- 基线KPI数据和改善目标

完成后粘贴链接，我将逐一审阅六维文档并生成完整诊断报告。
```

**等待用户提交飞书链接或内容后，进行诊断分析。**

---

## 诊断报告输出格式

收到用户信息后，按以下格式输出完整诊断报告：

### 【六维准备度评级结果】

```
① 战略价值度：X级（置信度：高/中/低）
   评级依据：[基于提交描述的具体说明，引用用户描述的关键数据]
   信息缺口影响：[若有缺口，说明对评级不确定性的影响]

② 数据准备度：X级（置信度：高/中/低）
   评级依据：[D1-D6各子维度的关键判定依据]
   信息缺口影响：[...]

③ 技术互联度：X级（置信度：高/中/低）
   评级依据：[...]

④ 工艺可控度：X级（置信度：高/中/低）
   评级依据：[...]

⑤ 组织能力度：X级（置信度：高/中/低）
   评级依据：[...]

⑥ 管理变革度：X级（置信度：高/中/低）
   评级依据：[...]
```

**评级标准（各维度1-5级）：**
- 参见支撑文件 [scoring-criteria.md](scoring-criteria.md)

### 【诊断结论】

```
准备度综合评分：X.X / 5
是否达到启动门槛：是/否（原因：...）

启动建议：
□ 全量启动：准备度充分，可按完整计划推进
□ 单台设备PoC：从1台设备小范围验证后再扩展
□ 先完成前置工作再启动：[具体前置工作清单，预计N个月]
```

### 【最关键瓶颈排序（5M1E根因分解）】

对最关键的2-3个瓶颈维度，进行5M1E根因分解：

```
瓶颈① [维度名称]（当前X级 → 需达Y级才能支撑目标）
对目标的影响：[不解决这个瓶颈，目标实现概率下降X%]
5M1E根因分解：
  人（Man）：[具体问题，如"AI工程师缺失，工艺专家知识未文档化"]
  机（Machine）：[如"设备无数字接口，只能手动读取参数"]
  料（Material）：[如"原料批次信息未关联生产数据"]
  法（Method）：[如"GR&R未做，测量系统可信度未知"]
  环（Environment）：[如"IT/OT隔离，数据无法实时传输"]

瓶颈② [维度名称]
...
```

### 【资料补充建议】

```
优先级1：补充[具体信息/文件]
  理由：将使[维度]评分置信度从低→高，可能将评级从X级提升至Y级
  预计收集难度：[低（现有文件整理）/ 中 / 高（需要新的测试/研究）]

优先级2：补充[...]

优先级3：补充[...]
```

### 【目标-现实校准评估】⚠️ 条件触发

**触发条件（满足任一条即输出此模块）：**
- 用户 1.3-B 雄心度分级为🟠激进 或🔴变革性
- 用户 OKR 信心分数 ≤ 4
- 综合准备度评分 < 2.5 且目标改善幅度 > 15%

**未触发时：** 标注"✅ 目标与准备度匹配，无需校准"即可。

**触发时输出：**

```
【目标-现实校准评估】⚠️

目标雄心度（来自1.3-B）：🟢保守/🟡适中/🟠激进/🔴变革性
当前综合准备度：X.X / 5级
支撑该目标所需最低准备度（基于WEF Lighthouse同类场景）：≥ X级
关键缺口维度：[差距最大的1-2个维度及缺口大小]

可达性判定：
□ ✅ 匹配：目标在当前准备度下，[X]个月内可合理期待实现
□ ⚠️ 偏激进：按当前准备度，预计仅能实现目标的[X%]，建议选择以下校准选项
□ ❌ 严重错位：当前准备度无法在设定时间内支撑该目标，必须修正

校准选项（⚠️或❌时必须提供三选一）：
  选项A · 调整目标值：
    → 匹配当前准备度（X级）的合理目标：___ / 预计[Y]个月内可达
    → 匹配Stage 1完成后准备度（X+1级）的合理目标：___ / 预计[Z]个月可达
  选项B · 扩展时间线：
    → 维持原目标，但时间节点延至___ / 需先完成[前置工作]约[N]个月
  选项C · 分阶段设目标（推荐优先考虑）：
    → 第一里程碑（90天）：___（低风险验证阶段，可见成效）
    → 第二里程碑（[6-12]个月）：___（规模化阶段）
    → 最终目标（[12-24]个月）：___（达成完整目标）
```

## Step 1.5：目标校准深度交互（仅当诊断报告判定为⚠️或❌时执行）

**方法论来源：** McKinsey "Ambition-Reality Calibration" · OKR渐进式拆解（Google re:Work）· Stage-Gate阶段性交付（Robert Cooper）

**触发条件：** Step 1诊断报告"目标-现实校准评估"判定为⚠️偏激进或❌严重错位。

用户确认选择（A/B/C）后，按以下步骤生成校准方案：

```
基于你给出的校准评估，目标存在[⚠️偏激进/❌严重错位]。

用户选择：[A/B/C]
补充约束（用户填写）：
• 不可改变的约束：[如"管理层已对外承诺年底目标"/"预算固定"/"时间节点不可变"]
• 可调整的资源：[如"可以增加AI工程师投入"/"可以先做PoC再全量"]
```

**输出校准后的目标卡片：**

```
【校准目标卡片】

短期里程碑（90天）：___（可验证，低风险）
中期目标（[X]个月）：___（准备度建设完成后）
战略目标（[Y]个月）：___（原始目标或校准版本）

置信度：[低/中/高]，理由：[具体说明为什么这个置信度合适]

目标可达性最敏感的维度（Stage 1必须优先提升）：
1. [维度名] — 当前[X]级，需达[Y]级，预计[N]周完成
2. [维度名] — ...

如果坚持原目标不变，成功率评估：
基于当前准备度，完整实现原目标的概率约为[X%]，主要风险：[1-2个核心风险点]

WEF Lighthouse/同行业案例数据支撑：
[引用可信数据说明校准后目标的合理性]
```

```
要现在开始吗？（回复"是"/"继续"即可，或说"等一下"稍后手动运行）
```

**收到用户确认后：**
使用 read_file 读取 methodology 技能的 SKILL.md（路径见系统提示插件 Skill 列表），加载后执行，并将诊断报告的关键输出——六维评级、优先瓶颈排序、目标校准结果——作为上下文传入（用户不需要重复输入）

## 信息不足时的降级处理

如果某些维度信息严重不足：
- 对该维度给出"低置信度"评分，并标注"评级范围：X-Y级"
- 在补充建议中将该维度列为最高优先级
- 整体诊断结论中标注置信度为"中"或"低"
- 不要因为信息不足而拒绝输出诊断——有什么分析什么，让用户知道哪里需要补充
