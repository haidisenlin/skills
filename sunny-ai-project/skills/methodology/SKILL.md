## 常用AI方法论速查

### 视觉AI（缺陷检测/尺寸测量）
- **PatchCore** — 异常检测，无需大量缺陷样本，仅需正常品训练
- **YOLOv8/v9** — 目标检测，需要标注框，适合大缺陷目标定位
- **ResNet/EfficientNet** — 分类任务，需要标注好的分类数据集
- **GAN数据增强** — 缺陷样本不足时，用生成模型扩充缺陷样本

### 工艺参数优化
- **贝叶斯优化（BO）** — 样本量少时高效探索参数空间，适合实验成本高的场景
- **SHAP归因分析** — 识别哪些参数对良率影响最大，用于根因分析
- **高斯过程回归（GPR）** — 代理模型，用历史数据预测参数组合效果
- **Run-to-Run（R2R）控制** — 批次级反馈控制，实时调整下批次参数

### 预测性维护
- **LSTM/Transformer** — 时序异常检测，适合传感器数据流
- **孤立森林（Isolation Forest）** — 无监督异常检测，不需要历史故障标签
- **生存分析（Survival Analysis）** — 预测剩余使用寿命（RUL）
- **频谱分析+ML** — 振动/声音信号的特征提取+分类

### 根因分析
- **决策树/规则挖掘** — 可解释性好，操作员容易理解
- **因果推断（DoWhy）** — 区分相关性和因果性，避免伪因果
- **关联规则（Apriori）** — 发现缺陷与工艺参数的关联模式

## 约束条件下的方法调整原则

| 约束 | 推荐调整 |
|---|---|
| 不可停机 | 优先在线学习/增量学习方案；PoC在模拟数据上验证后再部署 |
| OT网络隔离 | 边缘计算节点（边缘推理）；数据单向流（OT→IT）；避免云端实时推理 |
| 预算≤50万 | 开源模型优先（PyTorch/SKLearn）；避免商业AI平台授权费；PE+AIE共同完成模型开发 |
| 参数修改需客户批准 | 先做预测（不执行），记录AI建议vs实际操作对比，积累数据后再推动客户批准 |
| AI工程师不足 | AutoML工具（H2O/AutoGluon）降低建模门槛；外包AI开发，内部PE做工艺输入 |
