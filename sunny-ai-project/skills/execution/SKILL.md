# 执行管理框架（阶段 5）

**管理原则：** Agile-Stage-Gate × DMAIC × CRISP-DM三轨合一框架管理AI项目全生命周期。每阶段末设置阶段门控（Stage Gate），所有硬标准必须全部满足方可进入下一阶段。

**参考来源：** Robert G. Cooper《Stage-Gate Process》· PMI PMBOK 7th · Six Sigma DMAIC · CRISP-DM 1.0 · McKinsey Rewired（2023）· WEF Lighthouse Factory规模化路径

## 三轨框架概览

```
时间轴：    [  90天  ]────────[    6个月    ]────────[   12个月   ]────────[   24个月   ]
Stage-Gate：  Stage 1              Stage 2               Stage 3               Stage 4
DMAIC：       Define              Measure+Analyze         Improve               Control
CRISP-DM：   业务理解/数据理解    数据准备/建模/评估      部署/规模化           监控/新DMAIC周期
成熟度：      启动就绪             PoC验证                工业化部署            制度化+扩展
```

## Stage Gate门控评审标准

### Gate 1（Stage 1结束，约90天）

**硬标准（全部必须满足）：**
- [ ] **G1.1 数据可行性：** X-Y关联率≥60%（或有明确提升路径，Stage 2内可达80%+）
- [ ] **G1.2 测量可信度：** GR&R<30%（可接受）；或有明确改善计划
- [ ] **G1.3 设备接口：** 目标设备数据采集方案已确认（接口类型/采集频率/延迟）
- [ ] **G1.4 PoC方案立项：** PoC技术方案已通过Sponsor审批（算法/数据需求/成功标准）
- [ ] **G1.5 团队就绪：** PE+AIE+ITE三个核心角色已就位（内部或外包均可）

**如有目标校准结果，额外检查：**
- [ ] 校准后的阶段性目标已经管理层认可

### Gate 2（Stage 2结束，约6个月）

**硬标准：**
- [ ] **G2.1 PoC精度达标：** 验证集精度达到目标的≥70%
- [ ] **G2.2 数据管道稳定：** 自动数据采集运行≥2周无重大中断；延迟<指定阈值
- [ ] **G2.3 GR&R达标：** GR&R<30%（如Stage 1为"有改善计划"，此时必须达标）
- [ ] **G2.4 PoC实际效果：** 单台设备AI建议采纳率≥50%；可量化的KPI改善信号（哪怕微弱）
- [ ] **G2.5 规模化计划：** Stage 3全线推广计划（含培训/IT扩展/预算）已获批准

### Gate 3（Stage 3结束，约12个月）

**硬标准：**
- [ ] **G3.1 规模化覆盖：** 目标范围内设备覆盖率≥70%（已部署并在运行）
- [ ] **G3.2 持续采纳：** AI推荐采纳率连续4周≥70%（有日志数据支撑）
- [ ] **G3.3 模型稳定性：** PSI<0.25（无显著漂移）；过去1个月精度无超出阈值下降
- [ ] **G3.4 无重大损失：** 未发生因AI误判导致的重大质量事故
- [ ] **G3.5 ROI初步达成：** 实际KPI改善≥预期的60%；全年ROI测算为正

### Gate 4（Stage 4结束，约24个月）— 项目关闭标准

**硬标准：**
- [ ] **G4.1 ROI达成：** 24个月实际ROI达成率≥80%
- [ ] **G4.2 自主运营：** AI系统在无PM干预下稳定运行≥3个月（运营已交接给OPS）
- [ ] **G4.3 方法论可复制：** 第2个工序AI项目已正式立项，使用本项目方法论框架
- [ ] **G4.4 知识资产化：** 《AI调优Playbook》已发布并经实际验证；知识库≥20个案例
