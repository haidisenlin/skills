# 场景机会地图输出（Step B-1 ~ B-4）

3 轮深度信息（或用户选择跳过第 3 轮）收集完成后，按以下格式输出。

## 【Step B-1：痛点理解确认】

用 3-5 句话概括对核心痛点的理解，说明哪些信息有高置信度、哪些基于推断。

## 【Step B-2：AI 场景机会地图】

按"提质 → 提效 → 提敏捷"三维度分类，每个场景包含：

```
═══ 维度一：提质（Quality Improvement）═══

场景 Q1 ── [场景名称]
  └ AI类型：[视觉AI / 参数关联 / 根因分析等]
  └ 解决的痛点：[对应用户描述的哪个具体数字化问题]
  └ 预期价值：[年化收益量级估算]
  └ 数据可行性：[高/中/低，基于准备文档系统清单 + 第2轮数据现状]
  └ 技术复杂度：[低（成熟方案）/ 中 / 高]
  └ 启动周期：[PoC预计约X个月]
  └ 综合推荐度：⭐⭐⭐⭐⭐

═══ 维度二：提效（Efficiency Improvement）═══
[同格式]

═══ 维度三：提敏捷（Agility Improvement）═══
[同格式]
```

## 【Step B-3：Top 3 优先场景深度分析】

每个场景说明：推荐理由（价值/数据/可行性/启动门槛）+ 主要风险（1-2 个）

## 【Step B-4：第一个 AI 项目推荐 + 场景确认描述】

```
推荐场景：[场景名称]
推荐理由：[2-3句话]

── 场景确认描述（可直接传入下一个技能）──
• BU / 厂区 / 部门 / 角色：[来自准备文档解析结果]
• 行业上下文：舜宇光学 精密光电制造
• 改善领域：[对应A-K领域]
• 改善对象：[一句话场景描述]
• 核心痛点：[数字化的痛点描述]
• 项目授权类型：[①/②/③/④]
• Sponsor：[姓名/职位 或 "暂无"]
• 对AI的初步设想：[如"用视觉AI替代人工目检，识别XX类缺陷"]
```

## 信息不完整时的处理

- 基于已有信息先输出分析，置信度不足的地方标注「[低置信度：待补充]」
- 告知用户补充哪条信息可以最大幅度提高推荐精准度
- 不要因为信息不完整而阻塞输出

## 输出 Step B-4 后

进入 `references/workshop-w5-canvas.md`（Bernard Marr Canvas 把推荐场景包装成 30 秒电梯演讲）。
