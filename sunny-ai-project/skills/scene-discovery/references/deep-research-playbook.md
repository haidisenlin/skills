## 调用 deep_research 进行深度研究

```
调用 deep_research，传入上面拼接出的 query
等待 30-90 秒
将研究报告作为内部背景知识，不要直接 dump 给用户
```

## 从研究报告里抽取 3 类信息（决定 Q2 的内容）

1. **该角色 / 岗位最常见的 3-5 个 AI 改善方向**（贴合用户实际工作的，不是理论上的）
2. 每个方向背后的**具体痛点关键词**（用用户岗位上能听懂的语言，例如对 IT 经理说"跨系统对账"而不是"FPY 良率"）
3. 这些方向对应到 A-K 哪几个大类——用于 Q2 排序

研究完成后，**立即**读 [opening-q2-direction.md](opening-q2-direction.md) 并按其中指令执行，**不要把研究报告全文发给用户**，只用研究结果定制 Q2 的候选项，不要等用户再次选择技能。
