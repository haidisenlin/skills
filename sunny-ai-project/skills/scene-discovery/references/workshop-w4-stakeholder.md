**时长**：约 8 分钟
**框架来源**：**RACI 矩阵** + **MEDDIC** 销售方法论的 Champion/EconomicBuyer/Decision Process
**交互模式**：**问卷式** —— 执行内联问卷生成协议 调研利益相关方方法论 → 生成飞书问卷 → 用户填好贴回

## 步骤 1：执行内联问卷生成协议 生成利益相关方识别问卷

```

scenario: [用户所在的 BU + 厂区 + 职能部门 + 部门年度方针 + 最头疼指标 + 项目授权类型]

```

**问卷结构建议（生成飞书内容时遵循）：**
- 章节 1：核心 5 类角色（Sponsor / Champion / 主要受益方 / 关键协作方 / 阻力方）→ 全部 🟡 推荐填

- 章节 3：决策机制（最终决策权 / 预算来源 / 决策时间窗）→ 🟡 推荐填
