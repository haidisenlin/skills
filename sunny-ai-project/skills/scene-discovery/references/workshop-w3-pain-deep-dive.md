# Workshop W3：最头疼指标深挖

**时长**：约 8 分钟
**框架来源**：**5M1E 根因分析**（人/机/料/法/环/测量）+ **PDCA 循环**——找出"传统方法做不到、AI 才能解决"的具体卡点。
**AI 角色**：用追问法替代结对讨论。这是整个 workshop 里**最关键的一个模块**——找到"卡点"才能给出精准的 AI 推荐。

**⚠️ 交互模式：对话式（W3 唯一例外，保留追问）**

为什么 W3 不改成飞书问卷：根因挖掘的力量来自**追问**——用户说"应该是人的问题吧"，AI 反问"你有数据证明吗？还是经验判断？"；用户说"上次试过不行"，AI 反问"具体哪一条措施？什么原因不行？"。做成问卷一次性填完就失去了这个挖掘深度。W3 必须保持一次一问、AI 见到模糊答案就追问的节奏。

## 开场白

```
进入 workshop 第 3 个模块：最头疼指标深挖 🔥

这是整个 workshop 最重要的一个模块。我们要找到那个【传统方法做不到、
AI 才能解决】的具体卡点。

预计 8 分钟，我会用追问法帮你挖出根因。
```

## 第一问 — 选定指标

```
你在准备文档里填的 KPI 有：
  • {Metric 1}
  • {Metric 2}
  • {Metric 3}

哪一个是你最头疼的？为什么?
```

## 第二问 — 已尝试方案（必须先念回用户在方针表里写的「既定措施」）

```
针对【{用户选的指标}】，你在方针表里写的【既定措施】是：
  {对应 KPI 行的"既定措施"原文}

我现在要追问 2 件事：
  1. 这些既定措施目前实际进展怎么样？哪些已经在做、哪些还没启动？
  2. 已经做了的措施里，哪些有效、哪些没效？没效的为什么？

格式参考：
  既定措施 ①{XXX} → 进展：{已完成/进行中/未启动} → 结果：{有效/无效，原因}
  既定措施 ②{YYY} → ...

⚠️ 如果你方针表里没写措施 / 措施列空着，告诉我"没规划过措施"，
   这本身就是一个重要信号——意味着这条 KPI 还在"目标导向但缺路径"的状态。
```

**追问规则（AI 内部）**：
- 必须从 `KPI[选中行].既定措施` 槽位里取出原文念回，让用户感觉 AI 真的"读懂了我们的方针表"
- 如果某条措施用户说"试过没效果"，**那就是 AI 推荐的核心切入点**——AI 在最终输出里要明确说"传统措施 X 没解决，AI 从 Y 角度补位"
- 如果某条措施用户说"还没启动"，AI 不要推荐和它重复的方向，而要推荐"能加速这个既定措施落地的 AI 工具"

## 第三问 — 失败根因（5M1E 追问）

```
我注意到你说【{某个失败的方法}】没用。我用 5M1E 框架追问几个问题，
帮你找到真正的失败根因：

  Man（人）：操作人员的手法 / 经验 / 培训是否参与了这次失败？
  Machine（机）：设备的状态 / 参数 / 老化是否参与了？
  Material（料）：来料的批次 / 供应商 / 规格波动是否参与了？
  Method（法）：工艺方法 / SOP / 决策逻辑本身是否有问题？
  Environment（环）：温湿度 / 净化 / 照明 / 电源等环境因素？
  Measurement（测）：测量系统 / 数据采集 / 数据质量是否可信？

你认为哪 1-2 个最可能是真正的根因？为什么？
```

**追问规则（AI 内部）**：如果用户对失败根因模糊作答（"应该是人的问题吧"），继续追问："你说'应该是'——你有数据证明吗？还是这是经验判断？如果是经验判断，那这本身就是一个有趣的信号：传统方法依赖经验，AI 可以做的事情之一就是把经验数据化。"
