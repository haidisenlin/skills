# 开场 Q2：个性化改善方向

**必须基于上一步 deep_research 的结果生成，不要硬编码 11 类候选。**

## 发给用户的问题格式

```
基于你是【{BU} - {厂区} - {部门} - {角色}】，结合舜宇这类岗位的常见数字化痛点，
我给你筛选了几个最相关的 AI 改善方向（点选一个最主要的，后面还可以多选）：

🎯 高匹配（AI 推荐你优先选这几个）：
  {字母} - {大类名}：{用户角色能听懂的具体痛点示例 1}、{示例 2}、{示例 3}
  {字母} - {大类名}：{示例 1}、{示例 2}、{示例 3}
  {字母} - {大类名}：{示例 1}、{示例 2}、{示例 3}

📋 备选（如果以上都不是你想做的，也可以从这里选）：
  {剩余的 A-K 大类，只列字母 + 大类名，不列示例}
  K - 其他（请描述）
```

## 不同角色的 Q2 形态示例（仅供参考，**实际内容必须由 deep_research 结果驱动**）

### 示例 A：流信部 / IT / 数字化项目经理

```
🎯 高匹配：
  J - 数字化转型：MES↔ERP↔PLM 跨系统数据打通、低代码表单流转效率、RPA 替代手工对账、报表口径一致性治理
  B - 质量管理：跨厂区质量数据看板、SPC 自动化、客诉根因 NLP 分类
  I - 人才与组织：知识库智能问答、新员工 onboarding 路径自动化、培训内容个性化推送

📋 备选：A 生产效率 / C 设备可靠性 / D 精益流程 / E 供应链 / F 成本 / G 能源 / H 安全 / K 自定义
```

## 内部解析提示

「高匹配」3 个候选必须来自 deep_research 报告里出现频率最高 / 案例最多的方向，不能由 AI 凭空想象。如果研究报告对该岗位结果稀疏（如非常细分的角色），允许降级为 2 个高匹配 + 完整 A-K 备选。
