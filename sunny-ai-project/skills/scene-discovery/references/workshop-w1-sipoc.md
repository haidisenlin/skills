# Workshop W1：SIPOC 工作流程拆解（问卷式）

**时长**：约 10 分钟
**框架来源**：**SIPOC**（Supplier-Input-Process-Output-Customer），Six Sigma DMAIC「Define 阶段」标准工具
**交互模式**：**问卷式** —— 先执行内联问卷生成协议 做方法论调研，生成飞书问卷，用户填好贴回来

## 所有 workshop 模块的通用运行原则

- 用户填写完飞书文档链接或内容粘贴回来后，AI 基于内容做一次"理解复述 + 确认"
- 用户答得太抽象时（Process 只写"做产品"），AI 追问到具体例子
- 用户中途说"跳过这个模块"时，记录为「该模块未完成（置信度：低）」并继续
- **无必填字段**——所有字段都是 🟡 推荐填 或 ⚪ 选填，现场来不及填的可留空

## 步骤 1：生成 SIPOC 问卷 飞书问卷（内联协议）

⚡ **执行内联问卷生成协议**（见 `SKILL.md` 顶部「🧭 内联问卷生成协议」）。差异参数如下，执行完必须调用 `sunny_ai__feishu_create_doc` 并把返回链接发给用户：

```
topic: "Six Sigma DMAIC Define 阶段 SIPOC 工作流程拆解标准方法论"
scenario: [用户所在的 BU + 厂区 + 职能部门 + 角色 + 工作类型（现场/办公/混合）]
industry: 舜宇光学 精密光电制造
doc_name: "[厂区代码]-[姓名]-W1-SIPOC工作流程-[今日日期]"
purpose: "拆解用户工作流程 SIPOC 5 要素，特别是 Process 步骤要拆到 3-7 个具体动作，为后续 AI 场景识别定位价值链位置"
```

## 步骤 2：将 `sunny_ai__feishu_create_doc` 返回的飞书文档链接发给用户

前面加以下引导语：

```
现在进入 workshop 第 1 个模块：用 SIPOC 拆解你的工作流程 🔧

SIPOC 是六西格玛 Define 阶段的经典工具，5 个字母代表 5 个要素：
  Supplier（上游）→ Input（输入）→ Process（流程）→ Output（输出）→ Customer（下游）

我已经基于 Six Sigma 标准方法论 + 你的岗位特点，为你定制了一份飞书问卷。
请打开文档按字段填写，填完把链接粘贴回来。

📌 Process 部分是核心——请把你的工作拆成 3-7 个具体动作步骤：
   现场示例：清洗 → AA 对准 → 点胶 → 固化 → AOI 复检 → 包装
   办公示例：需求评审 → 技术方案设计 → 开发 → UAT → 上线 → 运维监控
   ❌ 不要写"做产品"或"处理需求"这种太粗的描述

预计 10 分钟。现场来不及填完的字段可留空，🟡 推荐填的缺了会影响后续
AI 场景推荐的精准度，但不阻塞。

━━━ 飞书文档 ━━━
{sunny_ai__feishu_create_doc 返回的文档链接}
```

## 步骤 3：等用户提交

- ① 用户粘贴飞书文档链接 → 调用 `sunny_ai__feishu_get_doc` 解析
- ② 用户直接粘贴文档内容 → 解析
- ③ 用户说"现场来不及细填，我口述" → 降级为对话式（按原来的 5 个问题顺序问）

## 步骤 4：理解复述 + 确认 + Process 追问

基于用户填写的内容，复述一次：

```
👌 我把你填的 SIPOC 整理一下：

  S → I → P → O → C
  {Supplier} → {Input} → 【{Process 3-7 步}】 → {Output} → {Customer}

价值链定位（基于 Porter Value Chain）：你属于【核心运营 / 物流供应链 /
支持职能-技术 / 支持职能-人才 / 支持职能-治理 / 支持职能-基础设施】中的
【{AI 判断的类别}】。

{如果 Process 只有 1-2 步或写得太抽象，这里追问：
"⚠️ 我看你的 Process 只写了【X 步/太抽象】——你能把【XX 步骤】再展开
 吗？这一步里你具体做哪几件事？这直接影响 AI 能不能找到你工作里的改善切入点。"}

如果有要修正的地方告诉我；没问题的话，我们进入第 2 个模块。
```

## 追问规则

如果用户填写的 Process **只有 1-2 步**或**写得太抽象**（如"做产品"/"处理需求"），必须追问到 3-7 个可识别的动作。追问话术："你能把【XX 步骤】再展开吗？这一步里你具体做哪几件事？"
