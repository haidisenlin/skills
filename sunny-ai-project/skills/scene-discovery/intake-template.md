## 📋 这份文档是什么 / 不是什么

✅ 这是一份 **Workshop 会前准备清单**。请在工位 / 家里花 **10-15 分钟** 把它填好，带到 workshop 会场。

❌ 这**不是**一份完整的 AI 场景发现材料。完整的诊断会在 workshop 会场**通过 AI 助手对话完成**（约 90 分钟），你只需要带这份准备好的文档过去就行。

## 🤖 你会在 Workshop 会场和 AI 助手做什么？

到了会场，你会打开 AI 助手（Claude Code），把这份准备文档的飞书链接贴给它。AI 会按以下顺序**用问答的方式**带你走完整个流程：

| 阶段 | AI 在做什么 | 时长 |
|---|---|---|
| **0. 解析准备文档** | AI 读取你的 3 块准备内容，回放它的理解让你确认 | ~3 分钟 |
| **1. 项目授权类型** | AI 问你 1 个简短问题：这次 AI 想法的来源是上级交办/自发探索/集团专项/客户驱动 | ~2 分钟 |
| **2. 定向深度研究** | AI 静默查行业最佳实践，根据你的角色定制候选清单 | ~1 分钟 |
| **3. 个性化改善方向** | AI 给你 3 个高匹配 AI 改善方向（不是泛泛的 11 类） | ~3 分钟 |
| **🎯 4. SIPOC 工作流程拆解** | AI 引导你用六西格玛 SIPOC 工具拆解你的工作 | ~10 分钟 |
| **🎯 5. GQM 关键问题识别** | AI 帮你从 Goal 推导出 2-3 个"可回答的关键问题" | ~5 分钟 |
| **🎯 6. 最头疼指标深挖** | AI 用追问法帮你找到"传统方法做不到、AI 才能解决"的卡点 | ~8 分钟 |
| **🎯 7. 数字化成熟度自评** | AI 引导你做 5 维成熟度自评，并模拟"互校准"追问 | ~10 分钟 |
| **🎯 8. 利益相关方校准** | AI 用 4 个扫描问题帮你发现"隐形利益相关方" | ~5 分钟 |
| **9. 三维痛点量化 / 数据可行性 / 改善历史** | AI 生成 3 份飞书文档模板让你深度填写 | ~30 分钟 |
| **10. 输出场景机会地图** | AI 输出 Top 3 推荐 + 第一个 AI 项目场景描述 | ~5 分钟 |
| **🎯 11. AI Use Case Canvas（基于 Top 3）** | AI 引导你用 Bernard Marr 一页画布捕捉你最想推进的那个场景 | ~10 分钟 |

> 🎯 标记的步骤是**原本需要人类 facilitator 现场带领的 workshop 环节**——现在由 AI 用问答形式完成。你不需要任何 facilitator 引导，AI 会一步步问你。

## 为什么这 3 部分要会前准备？

会前要准备的内容**只有 3 块**，因为它们要么需要**查系统**、要么需要**翻 OKR 文档**、要么需要**时间反思 + 查通讯录**——会场临时填会很仓促，影响后续 AI 推荐质量：

- ✅ **第一部分 · 个人定位**（5 字段，约 2 分钟）— 都是事实性字段
- ✅ **第二部分 · 你的核心目标 + KPI 数字**（约 8-10 分钟）— **必须查 MES / ERP / SAP / HR 系统**
- ✅ **第三部分 · 利益相关方草稿**（约 3 分钟）— 需要时间想 + 可能要查名单

> 💡 **填不全没关系**：如果某个字段你确实拿不到数据，写"待核实"或留空。重要的是**能填的尽量填好**，让会场时间用在更有价值的对话上。

## 第二部分 · 你的核心目标 + KPI 数字（必查系统）

> **框架来源**：**GQM = Goal-Question-Metric**（Basili & Rombach 1994）的 G 和 M 部分。Q（关键问题）会在 workshop 会场由 AI 引导你完成。
>
> **预计时间**：8-10 分钟（必须查系统）。
>
> **⚠️ 这是整份模板里最不能临时编的部分。** 请打开你日常用的 MES / ERP / SAP / HR 系统 / 飞书表格，把当前值和目标值的真实数字抄下来。会场临时填的"差不多 87%"和系统里实际的 86.4% 会让 AI 推荐的 ROI 估算偏差 30% 以上。

### 2.1 Goal — 你被考核的最高目标是什么？

> 用 1-2 句话写出来。如果你有多个并列目标，挑最重要的那个。
>
> **怎么写**：直接打开你的年度 OKR / 绩效目标文档抄过来。如果没有正式 OKR，写你年初向上级承诺的最重要目标。

```
[在此填写。示例：]

[现场示例] 在 2026 年内把车载镜头组装线 FPY 从 87% 提升到 92%，
            同时单位制造成本降低 8%。

[办公示例] 在 2026 年内把集团 HR 招聘流程的"岗位填充周期"从 45 天
            缩短到 25 天，同时离职率降至 12% 以下。
```

### 2.2 Metrics — 你被考核的核心指标（3-5 个）

> **必查系统**：每一行都要从真实系统里抄数字过来，不要凭印象。
>
> 如果某个指标你连数据源都不知道，写"数据源待查"——这本身就是一个有价值的诊断信号。

| 指标名称 | 单位 | 当前值 | 目标值 | 数据来源 | 复盘频率 |
|---|---|---|---|---|---|
| 例：FPY | % | 87.3 | 92 | MES 自动 | 周 |
| 例：岗位填充周期 | 天 | 45 | 25 | HR 系统 | 月 |
|  |  |  |  |  |  |
|  |  |  |  |  |  |
|  |  |  |  |  |  |

> 💡 关于"最头疼的指标 + 卡住原因 + 已经试过的方法"，**你不需要在这里写**——会场 AI 助手会用追问法帮你深挖（约 8 分钟），那时再讨论效果更好。

## 📚 完整方法论说明（如果你好奇）

整套 workshop 流程基于以下 6 个公认框架，AI 助手会在不同环节显式引用它们：

| 用在哪一步 | 框架 | 出处 |
|---|---|---|
| 准备文档 第二部分 + 会场 GQM Question | **GQM (Goal-Question-Metric)** | Basili & Rombach (1994), Software Engineering Lab |
| 准备文档 第三部分 + 会场利益相关方校准 | **RACI 矩阵** + **MEDDIC** | PMI 标准 + PTC 销售方法论 |
| 会场 SIPOC 工作流程拆解 | **SIPOC** | Six Sigma DMAIC「Define 阶段」标准工具 |
| 会场最头疼指标深挖 | **5M1E 根因分析** + **PDCA** | TPS 丰田生产体系 + Deming Cycle |
| 会场数字化成熟度自评 | **Gartner Digital Maturity** + **INCIT AIMRI** | Gartner 2024 + 国际制造业创新中心 |
| 会场 AI Use Case Canvas | **Bernard Marr AI Use Case Canvas** | 《Data Strategy》(Marr, 2021) |

> **核心理念**：你填得越认真，AI 给你的推荐就越精准——因为这些字段不是装饰，AI 在后续诊断时会显式引用它们的字段名。例如准备文档里的"最头疼指标"会成为 Top 3 推荐的优先级权重；会场 SIPOC 里填的 "Process 流程"会直接驱动 5M1E 痛点分解。
