# 场景预诊断（Step 0-A）

## 第一轮：了解场景基本情况

**如果用户来自场景发现（已有场景确认描述），直接跳到第二轮使用该描述。**

否则，**一次只问一个问题**，按以下顺序：

**问题1（先问）：**
```
你们公司是做什么行业/产品的？
改善领域大概属于哪类？（A-生产效率 / B-质量管理 / C-设备可靠性 / D-精益流程 / E-供应链 / F-成本管理 / G-能源 / H-安全 / I-人才 / J-数字化 / K-自定义）
```

**问题2（收到问题1答案后再问）：**
```
具体是哪个工序？打算用什么AI来改善？
```

**问题3（收到问题2答案后再问）：**
```
这个工序现在最大的痛点是什么？最好有具体数字（如"FPY 85%，目标92%"）
```

收到三个答案后，进入第二轮。

**K路径（自定义场景）**：如果用户选K，先判断最接近哪类管理改善领域，给出业界最权威的目标定义方法论（TQM/TPM/Six Sigma等），给出标准KPI（含公式和单位）、基线数据要求、世界级基准值，请用户确认方向后再进入第二轮。

---

## 第二轮：场景识别结果 + 启动飞书准备

基于第一轮信息，输出以下内容，然后**停下来等用户确认**：

```
【场景识别结果】

AI调优类型：[视觉AI / 工艺参数优化 / 预测性维护 / 流程分析 / 混合型]

核心技术挑战：[2-3句话，说明这个场景的主要难点]

最关键的准备度维度（按重要性排序）：
  第1位：[维度名] — [为什么对这个场景最关键]
  第2位：[维度名] — [原因]
  第3位：[维度名] — [原因]

---

📋 接下来我们将逐步填写六维飞书文档。

⚠️ 所有信息请通过飞书云文档录入，不要只在对话框里回复。
   原因：飞书文档可以多人协作、长期积累、AI重复引用——这是"文档飞轮"的基础。

飞书文档统一命名规则：
  [厂区代码]-[项目简称]-[维度名称]-[日期]
  例：YC-AOI视检-数据准备度-20260412

我们从【最关键维度】开始。准备好了吗？回复"开始"继续。
```

**步骤3** — 简单确认收到，告知下一个维度，问是否继续
