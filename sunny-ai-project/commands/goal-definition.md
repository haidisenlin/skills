## 第三步：目标合理性自检——雄心度-现实校准（1.3）⚠️ 核心防护网

这是最重要的防护环节。最常见的失败模式：管理层要求年底FPY从85%提到98%，但数据还没接通、MSA没做、AI工程师还没到位。

**A. 基础合理性核查（逐项询问）：**

| 自检项 | 用户回答 |
|---|---|
| 基线数据是实测值，不是估算 | 是/否 |
| 目标值参考过行业基准或竞争对手数据 | 是/否/不清楚 |
| 改善幅度在6-24个月内有行业先例可循 | 是/否（属于突破性目标）|
| 即使只实现60%的目标，ROI仍然为正 | 是/否/未估算 |
| 目标已经过有决策权的管理层正式认可 | 书面认可/口头认可/尚未沟通 |

> 如果有2项以上"否"，在诊断交互时重点验证目标合理性。

**B. 目标雄心度分级（必填）：**

引导用户计算改善幅度：
```
改善幅度 = (目标值 - 基线值) ÷ 基线值 × 100%
示例：FPY从85%→92% → 改善幅度 = (92-85)÷85×100% ≈ 8.2%
```

对照分级表确认雄心度：

| 改善幅度 | 行业数据支撑 | 雄心度分级 |
|---|---|---|
| < 10% | 制造AI第一年普遍可达 | 🟢 保守 |
| 10–25% | WEF Lighthouse工厂中位数 | 🟡 适中（推荐首选） |
| 25–40% | 需②③④三维度均达3-4级；典型周期18-24个月 | 🟠 激进 |
| > 40% | 仅顶尖10%灯塔工厂实现，需24-36个月 | 🔴 变革性 |

**C. OKR信心指数（必填）：**

问用户：**"如果让你打赌，在设定的时间节点内完整实现这个KPI目标的概率有多大（1-10分）？"**

| 信心分数 | 诊断信号 | 后续动作 |
|---|---|---|
| 8-10分 | 目标可能过于保守 | 考虑适当提高挑战性 |
| 5-7分 | ✅ 理想区间 | 正常推进 |
| 3-4分 | ⚠️ 目标偏激进 | 诊断交互后AI给出校准方案 |
| 1-2分 | ❌ 目标与现实严重错位 | 诊断交互后强制输出校准评估 |

**触发规则（满足任一条，诊断时触发专项校准）：**
- 雄心度分级为🟠激进或🔴变革性
- OKR信心分数≤4分
- 1.3-A有2项以上"否"
