  - OKR信心分数：___ / 10
  - 主要约束条件（不可停机/预算/参数权限等）

二、六维准备度资料汇总
  - ①战略价值度 飞书链接：
  - ②数据准备度 飞书链接：
  - ③技术互联度 飞书链接：

  - 六维评级（各维度 X级 / 综合 X.X/5）
  - 启动建议（全量/单台PoC/先完成前置工作）
  - 最关键瓶颈1/2/3：[维度名]

## 立项报告输出格式

收到用户信息后，生成完整立项报告草稿，包含以下章节：

```
【AI改善项目立项报告】草稿
版本：AI Draft V1.0　生成日期：[当前日期]　待团队审核确认

━━━ 封面 ━━━
项目名称：_____________
申请部门：_____________
项目经理：___________　所属工厂/事业部：___________
立项日期：___________　文档版本：V1.0（AI草稿）
机密等级：□公开 □内部（限项目团队）□保密（限管理层）

━━━ 审批记录 ━━━
| 审批层级 | 角色 | 姓名 | 审批意见 | 签字 | 日期 |
|---|---|---|---|---|---|
| 起草确认 | 项目经理（PM）| | □提交审批 | | |
| 一级审批 | 工厂/部门主管 | | □批准 □条件批准 □驳回 | | |
| 二级审批 | 工厂总经理 | | □批准 □条件批准 □驳回 | | |
| 三级审批 | CTO/首席数字官 | | □技术可行，批准 □需补充 □驳回 | | |
说明：预算<¥100万简化至二级；¥100-500万走三级；≥¥500万建议全部五级。

━━━ 第一章 执行摘要 ━━━

1.1 项目核心命题
[AI填写："通过在[工序/场景]部署[AI类型]，将[核心KPI]从[基线值]提升至[目标值]，
预计在[X]个月内为公司创造¥[N]万/年的持续收益。基于当前六维准备度综合[X]级，
项目具备启动条件，建议从Stage 1（90天定义与基础建设）开始推进。"]

1.2 关键数字一览
| 维度 | 数字 | 备注 |
|---|---|---|
| 核心KPI改善目标 | [填写] | 来自阶段1 |
| 年化损失（不改善）| ¥___万 | 当前与目标差距×年产量×单位价值 |
| 年化收益预期（基准）| ¥___万 | 来自ROI模型基准情景 |
| 项目总投资（范围估算）| ¥___万（±30%）| 待财务确认 |
| 预计投资回收期 | ___个月 | 基准情景 |
| 六维准备度综合评级 | ___级 | 来自Step 1诊断 |
| 项目启动结论 | □可直接启动 □条件启动 □建议延后 | AI判断 |

1.3 立项理由摘要
[AI填写3条：
1. 【战略对齐】本项目与公司[战略目标]直接挂钩，不行动的5年损失约¥___万；
2. 【经济可行】保守情景（60%价值实现）ROI仍为正，预计___个月收回投资；
3. 【技术可行】六维准备度综合___级，[最关键维度]已具备启动条件]

━━━ 第二章 项目背景与战略意义 ━━━
[基于用户提供的背景，描述行业背景、触发因素、现状痛点量化表、不改善的5年代价、战略对齐验证]

━━━ 第三章 六维准备度评估摘要 ━━━
[直接引用Step 1诊断结论，生成六维评级摘要表 + 启动建议]

━━━ 第四章 项目目标与范围 ━━━
[SMART目标表 + 范围说明（范围内/范围外）+ 主要约束]

━━━ 第五章 技术方案概述 ━━━
[AI技术选型 + 技术架构（边缘/云端/混合）+ 数据治理要点 + 自制vs外购建议]

━━━ 第六章 实施计划（里程碑摘要）━━━
[从Step 3行动计划提炼四阶段里程碑汇总]

| 阶段 | 时间节点 | 里程碑目标 | Gate标准（核心） |
|---|---|---|---|
| Stage 1（90天）| M+3 | [目标] | [核心门控标准] |
| Stage 2（6个月）| M+6 | [目标] | [核心门控标准] |
| Stage 3（12个月）| M+12 | [目标] | [核心门控标准] |
| Stage 4（24个月）| M+24 | [目标] | [核心门控标准] |

━━━ 第七章 项目组织与治理 ━━━
[核心角色表（Sponsor/PM/PE/AIE/ITE/QE/EE/OPS/CHG）+ 治理机制]

━━━ 第八章 预算（范围估算）━━━
[IT基础设施/硬件/AI软件/内部人力/培训/外部顾问/风险预备金 各Stage估算，标注"待财务确认"]

━━━ 第九章 风险评估与应对策略 ━━━
[Top 5项目风险摘要表 + 项目止损条件（数据不可行/技术不可行/ROI不可行/组织不可行）]

━━━ 第十章 投资效益分析（ROI模型）━━━
[预期收益量化表 + 三情景ROI分析（保守60%/基准100%/乐观120%）+ 累计现金流时序]

━━━

📝 草稿完成。需要用户审核补充的内容：
1. [第八章] 具体预算数字（各费用类型的真实报价）
2. [第七章] 团队成员实际姓名
3. [第十章] 财务部门确认的折现率和税后收益计算
4. [封面] 文件编号和机密等级
5. [审批页] 审批层级是否与公司内控制度一致

请将草稿中的 [AI填写：...] 标注全部用实际数据替换，然后提交审批。
```
