# 场景发现（Step 0-B）

## 问题分类规则

**直接对话（不需要飞书文档）：**
- 选择题、是/否确认、1-2句能回答清楚的简单问题

**必须通过 research-template 生成飞书文档：**
- 需要填写表格、量化数字的系统性分析
- 回答质量直接影响后续AI诊断精准度的关键问题

---

## 第零步：深度研究背景（开场问题之前，静默执行）

**在向用户提问之前，先调用 `deep_research` 工具。**

从用户的初始描述中提取行业背景 + 改善方向，拼接 query：
- 示例：用户说"舜宇流程信息部，质量和效率" → `"精密光学制造 IT流程信息化 质量管理效率提升 行业痛点 数字化最佳实践 典型案例 AI应用"`

调用 deep_research 工具，等待完成后将报告作为内部背景知识，**不对用户输出研究结果**，直接进入第一步。

## 第一步：两个选择题（直接对话，一次一个）

**问题1（先问）：**
```
你们是做什么行业/产品的？
（例如：光学器件/半导体封装/消费电子组装/汽车零部件/医疗器械等）
```

**问题2（收到问题1答案后再问）：**
```
你希望AI改善的方向大概属于哪个领域？

A - 生产效率：FPY/良率提升、OEE优化、产能瓶颈
B - 质量管理：缺陷率/PPM降低、客诉减少、SPC
C - 设备可靠性：非计划停机、预测性维护、MTBF提升
D - 精益流程：换型时间（SMED）、在制品压缩
E - 供应链与物料：OTIF、库存周转、供应商质量
F - 成本管理：单位制造成本、COPQ、物料损耗
G - 能源与可持续：单位能耗、碳排放
H - 安全与合规：TRIFR/LTIFR、安全检查
I - 人才与组织：人均效率、技能认证
J - 数字化转型：MES/AI应用、自动化率
K - 其他（请描述）
```

收到两个答案后，告知用户：
「很好！现在我需要更深入了解你的情况，以便精准识别最适合的AI场景。接下来有4轮信息收集，每轮我会为你生成一个结构化的飞书文档模板——填写越详细，AI推荐越精准。」

## 第六步：输出 AI 场景机会地图

四轮信息收集完成后，按以下格式输出：

### 【Step B-1：痛点理解确认】

用3-5句话概括对核心痛点的理解，说明哪些信息有高置信度、哪些基于推断。

### 【Step B-2：AI场景机会地图】

按"提质 → 提效 → 提敏捷"三维度分类，每个场景包含：

```
═══ 维度一：提质（Quality Improvement）═══

场景 Q1 ── [场景名称]
  └ AI类型：[视觉AI / 参数关联 / 根因分析等]
  └ 解决的痛点：[对应用户描述的哪个具体数字化问题]
  └ 预期价值：[年化收益量级估算]
  └ 数据可行性：[高/中/低，基于第3轮数据现状]
  └ 技术复杂度：[低（成熟方案）/ 中 / 高]
  └ 启动周期：[PoC预计约X个月]
  └ 综合推荐度：⭐⭐⭐⭐⭐

═══ 维度二：提效（Efficiency Improvement）═══
[同格式]

═══ 维度三：提敏捷（Agility Improvement）═══
[同格式]
```

### 【Step B-3：Top 3 优先场景深度分析】

每个场景说明：推荐理由（价值/数据/可行性/启动门槛）+ 主要风险（1-2个）

### 【Step B-4：第一个AI项目推荐 + 场景确认描述】

```
推荐场景：[场景名称]
推荐理由：[2-3句话]

── 场景确认描述（可直接传入下一个技能）──
• 公司/行业类型：[基于描述推断]
• 改善领域：[对应A-K领域]
• 改善对象：[一句话场景描述]
• 核心痛点：[数字化的痛点描述]
• 对AI的初步设想：[如"用视觉AI替代人工目检，识别XX类缺陷"]
```

要现在开始吗？（回复"是"/"继续"/"开始"即可，或说"等一下"稍后手动运行）

**收到用户任何肯定的回复后：**
使用 read_file 读取 commands/scene-prediagnosis.md，加载后执行，将已确认的场景描述作为上下文传入（用户不需要重复填写场景信息）。

## 信息不完整时的处理

- 基于已有信息先输出分析，置信度不足的地方标注「[低置信度：待补充]」

- 不要因为信息不完整而阻塞输出
